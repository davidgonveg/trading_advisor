                   ¡BIENVENIDO AL SISTEMA DE BACKTESTING!

Este sistema te permite probar tu estrategia de trading con datos históricos
de forma IDÉNTICA a cómo funciona en producción.

Si quieres empezar inmediatamente:

    1. cd /home/user/trading_advisor
    2. ./backtesting/quickstart.sh

Este script te guiará paso a paso verificando todo y ejecutando
un backtesting de prueba.


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Si quieres ejecutar tests para verificar que todo funciona:

    1. cd /home/user/trading_advisor
    2. python backtesting/test_system.py

Esto ejecutará 6 tests automatizados y te mostrará un resumen.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PASO 1: Verifica que tienes datos

    python3 << 'EOF'
    from database.connection import get_connection
    conn = get_connection()
    cursor = conn.cursor()
    cursor.execute("SELECT COUNT(*) FROM indicators_data")
    count = cursor.fetchone()[0]
    print(f"Datos disponibles: {count:,} filas")
    conn.close()
    EOF

    ✅ Si ves > 1000 filas, tienes suficiente
    ❌ Si ves 0 filas, ejecuta: python historical_data/populate_db.py


PASO 2: Ejecuta un backtesting de prueba

    python backtesting/run_backtest.py --symbol AAPL --days 30

    ⏱️  Esto tarda 30-60 segundos


PASO 3: Revisa los resultados

    cat $(ls -t backtesting/results/summary_*.txt | head -1)

    📊 Verás métricas como:
    - Return %
    - Win rate
    - Profit factor
    - Max drawdown
